Checkpoint-Driven Long Fiction Rewrites
Instead of losing progress to timeouts and usage limits, structure novel rewrites as checkpoint-based workflows where each chapter is committed individually. Claude can autonomously process chapters in sequence, saving state after each one, so interruptions never lose more than one chapter's work.
Getting started: Use TodoWrite to build a chapter manifest upfront, then have Claude commit after each chapter completes. Combine with a shell script that can resume from the last uncommitted chapter.
Paste into Claude Code:
Read all chapters in /chapters/ and create a TodoWrite checklist with one item per chapter. For each chapter: 1) Read the current version, 2) Rewrite it following these guidelines: [preserve atmospheric detail, expand rather than cut, keep character portrayals subtle not on-the-nose], 3) Write the new version to the same file, 4) Git commit with message 'Rewrite chapter N: [brief summary of changes]', 5) Mark the todo item complete. If you're interrupted, I can ask you to resume from the first unchecked todo item. Target 150% of original word count per chapter. Never delete texture or atmosphere — only add.
Parallel Agents for Multi-Deliverable Projects
Your most successful session used parallel agents to complete a story, an analysis page, and brainstorming items simultaneously. This pattern can scale — spawn dedicated agents for writing, building UI, and updating reference docs in parallel, with a coordinator agent merging results.

Getting started: Have Claude write a small validation script that checks word count deltas and grep for key terms, then iterate against it automatically before committing.
Paste into Claude Code:
Before rewriting any chapters, first create a validation script at /scripts/validate-rewrite.sh that: 1) Compares word count of new vs original (new must be >= 120% of original), 2) Checks that a list of key character names and location names from the original still appear in the rewrite, 3) Exits with error code and message if either check fails. Then for each chapter in /chapters/: read it, rewrite it with expanded scenes and preserved atmospheric detail, write the new version, run the validation script against old and new versions.
